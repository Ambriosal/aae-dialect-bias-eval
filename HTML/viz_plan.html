04 — Key Findings to Highlight
Finding 1 — Neutral is the universal blind spot

Every single model scores dramatically lower on neutral F1 than positive or negative. Range: 0.307 (DeepSeek) to 0.454 (Phi-3-Medium). Neutral AAE expression — matter-of-fact statements, dry humor, understated tone — is systematically misread. This is the register-flattening failure mode you identified in your qualitative analysis.

Finding 2 — Gemma has the worst bias despite 2nd-best accuracy

Gemma 3 is accurate 63% of the time but 78.5% of its errors are negative skew — the highest of any model. It's learning to be right by defaulting to negative. When it's wrong, it's almost always wrong in the same direction. High accuracy can mask severe directional bias.

Finding 3 — DeepSeek is the only model without a dominant error direction

DeepSeek shows 44% negative skew vs. 29.7% positive skew — the most balanced error distribution of the group, but also the worst macro F1 (0.503). It's uncertain rather than biased. A different failure mode worth calling out separately.

Finding 4 — The "positive → negative" misclassification is universal

Across all 6 models, the single most common error type is human said positive, model said negative. LLaMA: 125 instances. Gemma: 151. Mistral: 131. This directly replicates and extends the DGI finding from the original paper using human ground truth instead of SAE-translated labels.

05 — Visualization Plan (Priority Order)

🟢 Priority 1 — Core deliverables for Ryan (build first)

V1 · Confusion Matrices

6-Panel Confusion Matrix Heatmap

One 3×3 heatmap per model arranged in a 2×3 grid. Rows = human label, Cols = model prediction. Normalized by row (recall). The visual that immediately shows the negative-skew pattern to any reader.

seaborn.heatmap · matplotlib subplots
V2 · Model Ranking

Grouped Bar Chart: Accuracy + Macro F1

6 models on X-axis, two bars each (accuracy in blue, macro F1 in orange). Instantly communicates that high accuracy doesn't always mean low bias — the Gemma finding lives here.

matplotlib.pyplot.bar · grouped layout
V3 · Bias Direction

Diverging Bar Chart: Negative vs. Positive Error Skew

One bar per model. Left side = % errors skewing negative (red). Right side = % errors skewing positive (green). Center line = balanced. Makes the directional bias argument in one image — publishable as a standalone figure.

matplotlib horizontal barh · diverging scale
V4 · Per-Class F1

Grouped Bar: F1 by Sentiment Class per Model

3 bars per model (positive / neutral / negative F1). The neutral collapse across all models is the most striking pattern in the data and this is the clearest way to show it.

seaborn.barplot · hue by sentiment class

🟡 Priority 2 — Strong additions for the paper

V5 · Error Flow

Sankey / Alluvial Diagram: Human Label → Model Prediction

Flow diagram showing where each human label gets redirected by each model. Visually dramatic — the river of positive tweets flowing into negative classifications is immediately legible. One per model or combined.

plotly.graph_objects.Sankey · or matplotlib custom
★ Great resume piece
V6 · Radar Chart

Spider/Radar: Per-Class F1 Across Models

One polygon per model on a 3-axis radar (F1-positive, F1-neutral, F1-negative). Lets you compare "shape" of model performance — a model with uniform moderate F1 looks very different from one that's great at negative and terrible at neutral.

plotly or matplotlib polar axes
V7 · Label Distribution

Stacked Bar: Human Distribution vs. Each Model's Predictions

Shows not just accuracy but systematic over/under-prediction. If Gemma predicts negative 70% of the time but humans labeled it 36%, that's visible here even before looking at any accuracy metric.

matplotlib stacked bar · one bar per source
